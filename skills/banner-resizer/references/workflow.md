## Purpose

This skill lets Codex run the Banner Resize AI workflow directly for another user. The user provides a source banner and target sizes; Codex runs `scripts/resize_banner.py`; the script writes final resized PNG banners.

The workflow is:

```text
source banner
-> safe-area master generated by gpt-image-2
-> normalized safe-area image
-> Adobe Firefly Expand outpaint to final target canvas
-> exact-size PNG output
```

## Environment Variables

Use `.env.local`, `.env`, or shell exports. The script never stores secrets.

```bash
OPENAI_API_KEY=...
FIREFLY_CLIENT_ID=...
FIREFLY_CLIENT_SECRET=...
```

Adobe aliases are also accepted for notebook compatibility:

```bash
FIREFLY_SERVICES_CLIENT_ID=...
FIREFLY_SERVICES_CLIENT_SECRET=...
```

## Commands

Dry-run size and model planning:

```bash
python3 ~/.codex/skills/banner-resizer/scripts/resize_banner.py input/banner.png \
  --size 1200x628:1040x508 \
  --dry-run
```

Generate two outputs:

```bash
python3 ~/.codex/skills/banner-resizer/scripts/resize_banner.py input/banner.png \
  --size 1200x628:1040x508 \
  --size 1080x1080:960x960 \
  --output-dir output/banner-resizer \
  --zip
```

Load a custom env file:

```bash
python3 ~/.codex/skills/banner-resizer/scripts/resize_banner.py input/banner.png \
  --env-file .env.local \
  --size 970x250:890x190
```

## Workflow Stages

1. Parse every `--size` argument into target and safe-area dimensions.
2. Try safe-area attempts in this order:
   - requested safe area
   - midpoint between safe area and target canvas
   - full target canvas
3. Generate the safe-area master with `gpt-image-2` through the OpenAI Images Edit API using the source image and safe-area prompt.
4. Normalize the returned master to the used safe-area dimensions with center crop/resize.
5. If safe area equals target, save it as the final output.
6. Otherwise upload the normalized safe-area image to Adobe Firefly storage, call Firefly Expand, poll status if async, download the result, and resize to exact final target dimensions.

## Provider Notes

GPT Image 2:

- Uses `https://api.openai.com/v1/images/edits`.
- Enforces: both edges multiples of 16, edge length less than 3840, long:short ratio no more than 3:1, and total pixels from 655360 to 8294400.
- Uses the selected safe-area attempt when planning output size.

Adobe Firefly:

- Uses Adobe IMS client credentials to get an access token.
- Upload endpoint: `https://firefly-api.adobe.io/v2/storage/image`.
- Expand endpoint: `https://firefly-api.adobe.io/v3/images/expand-async`.
- Status polling accepts only HTTPS Adobe API/job hosts.

## Troubleshooting

- Missing `PIL` or `requests`: install `pillow requests python-dotenv`.
- Firefly auth failure: check `FIREFLY_CLIENT_ID` and `FIREFLY_CLIENT_SECRET`; do not use browser-only keys.
- Firefly expand returns no URL: inspect `manifest.json` and the printed provider response.
- GPT Image 2 rejects size: run with `--dry-run` and check the planned size.
- User asks for another generation or outpaint provider: explain that this skill intentionally uses only `gpt-image-2` plus Adobe Firefly outpaint.
