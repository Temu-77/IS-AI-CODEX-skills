  - 画像要素以外のオブジェクト要素
  - メインコピー
  - サブコピー
  - その他コピー要素
  - ロゴ
  - レイアウト
  - トーン&マナー
  - カラー
  - CTA周り
- ただし、文字の可読性が悪い場合には調整する。文字の大きさは12ポイント以上にする。

## `banner_concepts.json` の出力形式

4つのオブジェクトを持つJSON配列にする。キー名は機械処理用に英語のまま維持する。

```json
[
  {
    "concept_id": "concept_1",
    "title": "",
    "strategic_intent": "",
    "target_response": "",
    "main_copy": "",
    "sub_copy": "",
    "other_copy": "",
    "visual_direction": "",
    "layout_instruction": "",
    "asset_usage": "",
    "logo_usage": "",
    "tone_and_manner": "",
    "generation_prompt": "",
    "negative_constraints": []
  }
]
```

## 実行時の注意

- Node①-#7でコピーを作る場合、コピーは仮の入力ではなくバナー案オブジェクトの一部として扱う。
- ユーザーがコピーを指定した場合は、そのコピーを優先し、AIが勝手に別コピーへ置き換えない。
- ユーザーがコピー未指定の場合は、Node①-#7の原文意図に従って、各案のメインコピー/サブコピー/その他コピーをCodexが提案してよい。
- Node②の素材置換では、元バナーのコピー、ロゴ、レイアウト、CTA周りは固定要素として扱い、画像領域以外を変えない。
